# Garbage Collection Basics (GC 기본 원리)

## 1. 개념 정리
Garbage Collection(GC)은
JVM에서 더 이상 사용되지 않는 객체를 자동으로 식별하고
메모리에서 제거하는 메커니즘이다.

Java에서는 개발자가 직접 메모리를 해제하지 않으며,
GC가 Heap 영역을 중심으로 메모리를 관리한다.

GC의 핵심 개념은 다음과 같다.

- Reachable: 참조되고 있어 접근 가능한 객체
- Unreachable: 더 이상 참조되지 않는 객체
- GC Root: 객체 생존 여부 판단의 기준점

즉,
GC는 GC Root로부터 도달 가능한 객체만을 살아있는 객체로 판단한다.

---

## 2. 왜 중요한가
백엔드 서버는 장시간 실행되며,
요청 처리 과정에서 지속적으로 객체를 생성한다.

GC를 이해하지 못하면:
- OutOfMemoryError 발생 원인 파악 불가
- 응답 지연(STW) 원인 설명 불가
- 메모리 누수 판단 어려움
- JVM 튜닝 방향 설정 실패

특히 GC는
성능(Throughput)과 응답 지연(Latency)에 직접적인 영향을 준다.

---

## 3. 백엔드 실무 관점
GC는 Heap을 다음과 같이 관리한다.

- Young Generation
  - Eden
  - Survivor(S0, S1)
  - 단명 객체 중심
  - Minor GC 발생

- Old Generation
  - 오래 살아남은 객체 저장
  - Major GC / Full GC 발생

GC 동작 흐름은 다음과 같다.

1. 객체는 Eden에 생성
2. Eden이 가득 차면 Minor GC 수행
3. 살아남은 객체는 Survivor 영역으로 이동
4. 여러 번 살아남은 객체는 Old 영역으로 승격
5. Old 영역이 가득 차면 Major / Full GC 발생

실무에서는:
- Minor GC는 빈번하지만 짧게
- Full GC는 드물지만 길게

발생하는 것이 이상적인 상태다.

---

## 4. 트레이드오프 / 주의사항
- GC 수행 중에는 Stop-The-World(STW) 발생
- Full GC는 응답 지연을 크게 유발
- 객체 생존 시간이 길수록 GC 비용 증가
- 불필요한 객체 생성은 GC 부담 증가

즉,
> GC는 없앨 수 있는 대상이 아니라,
> 영향을 최소화하도록 설계해야 한다.

라는 트레이드오프가 존재한다.

## 5. 예상 면접 질문 & 답변 연습

### Q1. GC란 무엇인가요?
A.
GC는 JVM에서 더 이상 사용되지 않는 객체를 자동으로 정리해
Heap 메모리를 관리하는 메커니즘입니다.

---

### Q2. GC는 어떤 기준으로 객체를 제거하나요?
A.
GC Root로부터 도달 가능한지 여부를 기준으로,
도달 불가능한 객체를 제거합니다.

---

### Q3. Minor GC와 Full GC의 차이는 무엇인가요?
A.
Minor GC는 Young 영역을 대상으로 자주 발생하며 짧고,
Full GC는 Old 영역까지 포함해 발생하며 응답 지연이 큽니다.

---

## 6. 한 문장 요약 (면접용)
GC는
사용되지 않는 객체를 자동으로 정리해
Heap 메모리를 관리하는 JVM 핵심 메커니즘이며,
성능과 응답 지연에 직접적인 영향을 준다.
