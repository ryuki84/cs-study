# Garbage Collection Algorithms (GC 알고리즘)

## 1. 개념 정리
GC 알고리즘은
Heap 메모리에서 가비지 객체를 제거하기 위해
JVM이 사용하는 수집 방식이다.

GC 알고리즘에 따라
Stop-The-World 시간,
처리량(Throughput),
응답 지연(Latency)
특성이 달라진다.

주요 GC 알고리즘은 다음과 같다.

- Serial GC
- Parallel GC
- CMS
- G1 GC
- ZGC (개념 수준)

---

## 2. 왜 중요한가
GC 알고리즘 선택은
백엔드 서버의 성능 특성을 결정한다.

잘못된 알고리즘 선택은:
- 긴 GC Pause
- 응답 지연 증가
- 처리량 저하
- 장애 발생 가능성

으로 이어질 수 있다.

즉,
GC 알고리즘은
서버 성능 튜닝의 핵심 요소다.

---

## 3. 백엔드 실무 관점
각 GC 알고리즘의 특징은 다음과 같다.

### Serial GC
- 단일 스레드로 GC 수행
- STW 시간이 김
- 구조 단순
- 소규모 애플리케이션에 적합

### Parallel GC
- 여러 GC 스레드 사용
- 처리량 중심
- STW 존재
- 배치성 작업에 적합

### CMS (Concurrent Mark Sweep)
- Old 영역 대상
- STW 최소화 목표
- 동시 GC 수행
- 단편화 문제
- 최신 JVM에서는 사용 감소

### G1 GC
- Heap을 Region 단위로 관리
- 예측 가능한 GC 시간
- STW 최소화
- 대규모 Heap에 적합
- 최근 JVM의 기본 GC

### ZGC
- 초저지연 GC
- 매우 짧은 STW
- 대규모 Heap 지원
- 최신 JVM에서 사용
- 개념 수준 이해만 요구됨

실무에서는:
- 대부분 G1 GC
- 응답 지연이 매우 민감하면 ZGC 고려

하는 경우가 많다.

---

## 4. 트레이드오프 / 주의사항
- Serial / Parallel은 STW가 큼
- CMS는 단편화 문제 존재
- G1은 설정에 따라 성능 편차 발생
- ZGC는 최신 JVM 환경 필요

즉,
> GC 알고리즘은 만능이 없으며,
> 서비스 특성에 맞게 선택해야 한다.

라는 트레이드오프가 존재한다.

---

## 5. 예상 면접 질문 & 답변 연습

### Q1. GC 알고리즘을 왜 선택해야 하나요?
A.
알고리즘에 따라
응답 지연과 처리량 특성이 달라지기 때문에,
서비스 특성에 맞는 선택이 필요합니다.

---

### Q2. G1 GC를 많이 사용하는 이유는 무엇인가요?
A.
Heap을 Region 단위로 관리해
예측 가능한 GC 시간을 제공하고,
대규모 Heap에서도 안정적인 성능을 제공하기 때문입니다.

---

### Q3. ZGC는 언제 고려하나요?
A.
GC 지연 시간이 매우 민감한 서비스에서,
최신 JVM 환경을 사용할 수 있을 경우 고려합니다.

---

## 6. 한 문장 요약 (면접용)
GC 알고리즘은
응답 지연과 처리량 특성을 결정하는 요소이며,
실무에서는 G1 GC가 가장 일반적으로 사용된다.
