# TCP vs UDP

## 1. 개념 정리
TCP(Transmission Control Protocol)는 연결 지향 프로토콜로,
통신 전에 연결을 맺고 데이터 전송의 신뢰성과 순서를 보장한다.
패킷 손실이 발생하면 재전송을 수행하며,
흐름 제어와 혼잡 제어 기능을 통해 안정적인 통신을 제공한다.

UDP(User Datagram Protocol)는 비연결 프로토콜로,
연결 과정 없이 데이터를 전송하며
전송 순서나 데이터 도착 여부를 보장하지 않는다.
재전송, 흐름 제어, 혼잡 제어 기능이 없기 때문에
오버헤드가 적고 전송 속도가 빠르다.

즉,
- TCP는 신뢰성을 우선하는 프로토콜
- UDP는 속도와 실시간성을 우선하는 프로토콜이다.

---

## 2. 왜 중요한가
백엔드 서버는 대부분 네트워크 통신을 기반으로 동작하며,
프로토콜 선택에 따라 시스템 특성이 크게 달라진다.

TCP와 UDP의 차이를 이해하지 못하면:
- 데이터 유실로 인한 장애
- 응답 지연 증가
- 불필요한 서버 리소스 소모
- 잘못된 아키텍처 선택

과 같은 문제가 발생할 수 있다.

특히 금융 시스템이나 API 서버처럼
데이터 정합성과 신뢰성이 중요한 환경에서는
프로토콜 선택이 곧 서비스 안정성과 직결된다.

---

## 3. 백엔드 실무 관점
일반적인 백엔드 서버(Spring 기반 API 서버)는
HTTP/HTTPS 통신을 사용하며,
이는 TCP 위에서 동작한다.

그 이유는:
- 요청과 응답 데이터의 정확성 보장
- 패킷 순서가 중요한 비즈니스 로직
- 손실 발생 시 재전송을 통한 안정성 확보

때문이다.

반면 UDP는:
- 실시간 스트리밍
- 온라인 게임
- 음성/영상 통신

과 같이,
일부 데이터 손실을 허용하더라도
지연을 최소화해야 하는 영역에서 사용된다.

---

## 4. 트레이드오프 / 주의사항
- TCP는 신뢰성이 높지만,
  연결 설정 과정과 재전송, 제어 로직으로 인해
  지연과 오버헤드가 발생한다.
- UDP는 빠르지만,
  데이터 손실과 순서 뒤바뀜을 애플리케이션이 직접 처리해야 한다.

즉,
> 안정성을 선택하면 성능 비용을 감수해야 하고,
> 성능을 선택하면 신뢰성을 직접 책임져야 한다.

라는 명확한 트레이드오프가 존재한다.

---

## 5. 예상 면접 질문 & 답변 연습

### Q1. TCP와 UDP의 가장 큰 차이는 무엇인가요?
A.
TCP는 연결 지향 프로토콜로 데이터 전송의 신뢰성과 순서를 보장하고,
UDP는 비연결 프로토콜로 빠르지만 신뢰성과 순서 보장은 제공하지 않습니다.

---

### Q2. HTTP는 왜 UDP가 아니라 TCP를 사용하나요?
A.
HTTP 통신에서는 요청과 응답 데이터의 정확성과 순서가 중요하기 때문에,
재전송과 순서 보장을 제공하는 TCP가 적합합니다.

---

### Q3. UDP가 TCP보다 빠른 이유는 무엇인가요?
A.
UDP는 연결 설정 과정과 재전송, 흐름 제어가 없기 때문에
프로토콜 오버헤드가 적고 전송 지연이 최소화됩니다.

---

## 6. 한 문장 요약 (면접용)
TCP는 데이터 정합성과 신뢰성을 보장하는 프로토콜이고,
UDP는 일부 손실을 감수하고 속도와 실시간성을 우선하는 프로토콜이다.
