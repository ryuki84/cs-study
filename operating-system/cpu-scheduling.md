# CPU 스케줄링(CPU Scheduling)

## 1. 개념 정리
CPU 스케줄링이란 **여러 프로세스 또는 스레드 중  
어떤 작업에 CPU를 할당할지 결정하는 정책**이다.

CPU는 한 순간에 하나의 작업만 실행할 수 있기 때문에  
효율적인 스케줄링이 필수적이다.

---

## 2. 주요 스케줄링 방식
- FCFS(First Come First Served)
- SJF(Shortest Job First)
- Round Robin
- Priority Scheduling

현대 운영체제는 보통 **선점형 스케줄링**을 사용한다.

---

## 3. 왜 중요한가
CPU 스케줄링은 다음 지표에 직접적인 영향을 준다.
- 응답 시간
- 처리량
- 대기 시간

잘못된 스케줄링은:
- 응답 지연
- CPU 사용률 저하
- 컨텍스트 스위칭 증가

를 유발한다.

---

## 4. 백엔드 실무 관점
백엔드 서버에서는:
- 스레드 수
- CPU 코어 수
- I/O 비중

이 CPU 스케줄링 효과에 큰 영향을 준다.

CPU Bound 작업이 많은데 스레드를 과도하게 늘리면  
컨텍스트 스위칭 비용만 증가한다.

---

## 5. 트레이드오프 / 주의사항
- 스레드 수 ↑ → 동시성 ↑ / 컨텍스트 스위칭 ↑
- 스레드 수 ↓ → 응답 지연 가능성 ↑

즉,
> **CPU 코어 수와 작업 특성에 맞는 설정이 핵심**

이다.

---

## 6. 예상 면접 질문 & 답변 연습

### Q1. CPU 스케줄링이란 무엇인가요?
A.  
여러 작업 중 어떤 작업에 CPU를 할당할지 결정하는 정책입니다.

---

### Q2. 스레드를 많이 만들면 성능이 좋아지나요?
A.  
아닙니다. CPU 코어 수를 초과하면 컨텍스트 스위칭 비용이 증가해  
오히려 성능이 저하될 수 있습니다.

---

## 7. 한 문장 요약 (면접용)
CPU 스케줄링은 응답 시간과 처리량을 결정하며,  
백엔드 서버에서는 **스레드 수와 CPU 코어 수의 균형**이 중요합니다.
