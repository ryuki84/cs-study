# 가상 메모리(Virtual Memory)

## 1. 개념 정리
가상 메모리는 **물리 메모리보다 큰 메모리 공간을  
사용할 수 있도록 보이게 하는 운영체제 기법**이다.

프로세스는 실제 물리 메모리 주소가 아닌  
가상 주소를 사용한다.

---

## 2. 페이징과 페이지 폴트
- 메모리는 페이지 단위로 관리된다
- 필요한 페이지가 메모리에 없으면 페이지 폴트 발생
- 디스크에서 메모리로 적재

이 과정은 **큰 비용**이 발생한다.

---

## 3. 왜 중요한가
페이지 폴트가 자주 발생하면:
- 디스크 I/O 증가
- 응답 시간 급증
- 시스템 성능 급락

심한 경우 **OOM(Out Of Memory)** 상황으로 이어질 수 있다.

---

## 4. 백엔드 실무 관점
- JVM Heap 크기 설정
- 메모리 누수
- 과도한 객체 생성

모두 가상 메모리와 밀접한 관련이 있다.

OS 레벨에서 메모리가 부족해지면  
GC 성능 저하 → 전체 서비스 성능 저하로 이어진다.

---

## 5. 트레이드오프 / 주의사항
- 스왑(Swap)은 응급 처치일 뿐 근본 해결책은 아니다
- 메모리 사용 패턴 분석이 중요하다

---

## 6. 예상 면접 질문 & 답변 연습

### Q1. 가상 메모리란 무엇인가요?
A.  
물리 메모리보다 큰 메모리를 사용하는 것처럼 보이게 하는  
운영체제의 메모리 관리 기법입니다.

---

### Q2. 페이지 폴트가 자주 발생하면 어떤 문제가 있나요?
A.  
디스크 I/O가 증가해 응답 시간이 느려지고  
전체 시스템 성능이 급격히 저하됩니다.

---

## 7. 한 문장 요약 (면접용)
가상 메모리는 효율적인 메모리 관리를 가능하게 하지만,  
페이지 폴트가 잦아지면 성능에 치명적인 영향을 줍니다.
